MCP client

skawld can consume tools from external MCP servers (Model Context Protocol). v1 is client-only and supports stdio (local child process) and Streamable HTTP transports, and MCP tools (not resources or prompts). The config shape mirrors the Claude Agent SDK, so mcpServers objects are copy-paste compatible.

The ergonomic path is the mcpServers option on AgentOptions (re-exported from the main skawld entry). Servers connect lazily on the first session() call and disconnect on agent.close():

import { Agent } from "skawld";
import { AnthropicProvider } from "skawld/providers";

const agent = new Agent({
  provider: new AnthropicProvider(),
  model: "claude-opus-4-5",
  mcpServers: {
    github: { command: "npx", args: ["-y", "@mcp/github"], env: { GITHUB_TOKEN: "…" } }, // stdio
    docs:   { type: "http", url: "https://example.com/mcp", headers: { Authorization: "Bearer …" } },
  },
});

const session = await agent.session(); // connects MCP servers here
// MCP tools appear to the model as mcp__github__*, mcp__docs__*
await agent.close();                   // disconnects, kills stdio children

For custom registries or explicit lifecycle control, use the lower-level helper from skawld/tools:

import { connectMcpServers } from "skawld/tools";
import { defaultTools } from "skawld";

const mcp = await connectMcpServers({ github: { command: "npx", args: ["-y", "@mcp/github"] } });
const tools = defaultTools();
for (const t of mcp.tools) tools.register(t);
// …pass `tools` to new Agent({ ..., tools })…
await mcp.close();
  • Naming. Tools are namespaced mcp__<server>__<tool> (characters outside [a-zA-Z0-9_-] become _). This avoids collisions with built-ins and makes permission ToolRules match by exact name, e.g. { kind: "tool", tool: "mcp__github__create_issue", decision: "allow" }.
  • Permission scope. A tool's scope derives from its MCP readOnlyHint annotation: readOnlyHint: true → read (auto-allowed in default mode, runs in parallel); otherwise write (prompts via canUseTool in default mode, runs sequentially).
  • Validation. Input is passed through to the server, which validates against its own schema.
  • Results. Text and image content map directly; audio and embedded-resource blocks degrade to a text placeholder. A server-reported error becomes an is_error tool result.
  • Failure. If any configured server fails to connect, connectMcpServers (and therefore the first session()) throws and tears down any partially-opened connections.

Minimal usage

import { Agent, defaultTools } from "skawld";
import { AnthropicProvider } from "skawld/providers";

const agent = new Agent({
  provider: new AnthropicProvider(),       // reads ANTHROPIC_API_KEY
  model: "claude-sonnet-4-6",
  tools: defaultTools(),
  permissions: { mode: "default" },        // will prompt via canUseTool for writes/exec
});

const session = await agent.session();
for await (const event of session.run("write a hello world script in python")) {
  if (event.type === "assistant") {
    for (const block of event.message.content) {
      if (block.type === "text") process.stdout.write(block.text);
    }
  }
  if (event.type === "result") break;
}

await agent.close();

Switching providers

import { Agent, defaultTools } from "skawld";
import { OpenAIChatCompletionsProvider, OpenAIResponsesProvider } from "skawld/providers";

// OpenAI Chat Completions (works for OpenAI and OpenAI-compatible endpoints).
const a = new Agent({
  provider: new OpenAIChatCompletionsProvider(),
  model: "gpt-5",
  tools: defaultTools(),
});

// OpenAI Responses API.
const b = new Agent({
  provider: new OpenAIResponsesProvider({ reasoning: "medium" }),
  model: "gpt-5",
  tools: defaultTools(),
});

// An OpenAI-compatible endpoint (Ollama, vLLM, etc.).
const c = new Agent({
  provider: new OpenAIChatCompletionsProvider({
    baseURL: "http://localhost:11434/v1",
    apiKey: "ollama",
    contextWindowOverride: (m) => m === "llama3.1" ? 128_000 : undefined,
  }),
  model: "llama3.1",
  tools: defaultTools(),
});

Permissions with rules and a callback

import { Agent, defaultTools } from "skawld";
import { AnthropicProvider } from "skawld/providers";
import type { CanUseTool, PermissionRule } from "skawld/permissions";
import readline from "node:readline/promises";

const rules: PermissionRule[] = [
  { kind: "bash", pattern: "git status", decision: "allow" },
  { kind: "bash", pattern: "git diff",   decision: "allow" },
  { kind: "bash", pattern: { regex: "\\brm\\b" }, decision: "deny" },
  { kind: "path", paths: ["!/Users/me/repo/**"], decision: "deny" },
];

const rl = readline.createInterface({ input: process.stdin, output: process.stdout });
const canUseTool: CanUseTool = async (req) => {
  const answer = await rl.question(`\n[${req.tool_name}] ${req.summary}\nApprove? (y/N) `);
  return answer.trim().toLowerCase() === "y"
    ? { behavior: "allow" }
    : { behavior: "deny", message: "user declined" };
};

const agent = new Agent({
  provider: new AnthropicProvider(),
  model: "claude-sonnet-4-6",
  tools: defaultTools(),
  permissions: { mode: "default", rules, canUseTool },
});

Cancelling a run

const session = await agent.session();
const controller = new AbortController();

// Cancel after 30 seconds.
const timer = setTimeout(() => controller.abort(new Error("timeout")), 30_000);

try {
  for await (const event of session.run("refactor the parser", { signal: controller.signal })) {
    if (event.type === "result") break;
  }
} finally {
  clearTimeout(timer);
}

Equivalent via the session API:

setTimeout(() => session.abort(new Error("timeout")), 30_000);

Resuming a session

// First run.
const a = await agent.session({ meta: { title: "parser refactor" } });
for await (const _ of a.run("explain the parser architecture")) {}

// Later — same Agent or a different one, as long as the SessionStore is wired up.
const b = await agent.session({ id: a.id });
for await (const event of b.run("now implement the changes we discussed")) {
  if (event.type === "result") break;
}

Subclassing a built-in tool

import { Agent, defaultTools } from "skawld";
import { BashTool } from "skawld/tools";

class SafeBashTool extends BashTool {
  validate(raw) {
    const input = super.validate(raw);
    if (/\brm\b/.test(input.command)) {
      throw new Error("rm is disallowed by SafeBashTool");
    }
    return input;
  }
}

const tools = defaultTools();
// Replace the default BashTool with the safe variant.
tools.register(new SafeBashTool());  // ToolRegistry.register throws on duplicate; remove first if needed.

Note: v1's ToolRegistry.register throws on duplicate name. To replace a built-in tool, build the registry from scratch rather than using defaultTools(), or use tools.unregister(name) if you add that helper.

Custom compaction

import { Agent } from "skawld";
import type { CompactionStrategy } from "skawld";       // re-export forthcoming
// or: import type { CompactionStrategy } from "skawld/sdk";

const compaction: CompactionStrategy = {
  id: "drop-thinking-blocks",
  async compact({ messages }) {
    // Trivial: strip thinking blocks from older messages to save tokens.
    return messages.map((m, i) => i < messages.length - 10
      ? { ...m, content: m.content.filter(b => b.type !== "thinking") }
      : m);
  },
};

const agent = new Agent({ /* ... */ compaction });

Whether CompactionStrategy is re-exported from the main entry or only from skawld/sdk is an implementation detail; it should be available from skawld for ergonomic reasons. Add it to src/sdk.ts.

Event consumption patterns

Just the final text

let final = "";
for await (const event of session.run(prompt)) {
  if (event.type === "result") final = event.final_text ?? "";
}

Streaming tokens to a UI

const session = await agent.session();
const agentWithPartials = new Agent({ /* ... */ includePartialMessages: true });
const s2 = await agentWithPartials.session();

for await (const event of s2.run(prompt)) {
  if (event.type === "partial_assistant" && event.delta.kind === "text") {
    ui.appendStreaming(event.delta.text);
  } else if (event.type === "assistant") {
    ui.commitMessage(event.message);
  }
}

Cost tracking

let totalIn = 0, totalOut = 0, totalCache = 0;
for await (const event of session.run(prompt)) {
  if (event.type === "usage") {
    totalIn += event.usage.input_tokens;
    totalOut += event.usage.output_tokens;
    totalCache += event.usage.cache_read_tokens ?? 0;
  }
  if (event.type === "result") {
    // event.total_usage has the same numbers cumulatively.
  }
}

Error handling

Most errors arrive as ErrorEvent in the stream, followed by a ResultEvent with subtype: "error". The async iterator does not throw for normal operational errors. It will throw only for catastrophic conditions (e.g. provider misconfiguration detected at construction, abort signal fired before iteration started).

import { AuthError, RateLimitError, ContextLengthError } from "skawld";

try {
  for await (const event of session.run(prompt)) {
    if (event.type === "error") {
      if (event.error.name === "AuthError") { /* surface auth UI */ }
      if (event.error.name === "RateLimitError") { /* backoff banner */ }
    }
    if (event.type === "result") break;
  }
} catch (err) {
  // Catastrophic. err is a SkawldError subclass.
  if (err instanceof AuthError) { /* ... */ }
}

TypeScript configuration for consumers

Consumers need "module": "NodeNext" (or "Bundler") and "moduleResolution": "NodeNext" (or "Bundler") for ESM subpath exports to resolve. The package ships with declaration maps so go-to-definition works through the subpath barriers.

No commonjs build

v1 is ESM-only. CommonJS consumers can use dynamic import(); we do not ship a dual build. This keeps the package small and avoids the "two installs" footgun.
