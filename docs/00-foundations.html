Foundations

Shared types, the event union, and the typed error hierarchy. Everything else in skawld depends on the names defined here. Implement this module first.

This module is pure types and small helpers — no runtime logic worth speaking of, no I/O, no provider-specific concerns. It lives in src/core/ and is the single source of truth for the vocabulary used across the codebase.

Message and content shapes

skawld speaks a single normalized message shape internally. Providers translate to and from this shape at the wire layer. The normalized shape is closer to the Anthropic format than the OpenAI format because the Anthropic format expresses tool calls and tool results more cleanly as content blocks (which makes the loop simpler to reason about).

// src/core/types.ts

/** A single message in a conversation. */
export interface Message {
  role: "user" | "assistant";
  content: ContentBlock[];
}

/** Discriminated union of content blocks. */
export type ContentBlock =
  | TextBlock
  | ToolUseBlock
  | ToolResultBlock
  | ThinkingBlock
  | ImageBlock;

export interface TextBlock {
  type: "text";
  text: string;
}

export interface ToolUseBlock {
  type: "tool_use";
  /** Provider-assigned id, opaque to skawld. */
  id: string;
  /** Tool name (must match a registered tool). */
  name: string;
  /** Parsed JSON arguments. */
  input: Record<string, unknown>;
}

export interface ToolResultBlock {
  type: "tool_result";
  /** Must match the id of a preceding tool_use. */
  tool_use_id: string;
  /** Stringified output, or structured blocks (text + image). */
  content: string | Array<TextBlock | ImageBlock>;
  /** True when the tool execution itself errored. */
  is_error?: boolean;
}

export interface ThinkingBlock {
  type: "thinking";
  thinking: string;
  /** Some providers return a signature/identifier to echo back. Opaque to skawld. */
  signature?: string;
}

export interface ImageBlock {
  type: "image";
  source:
    | { kind: "base64"; media_type: string; data: string }
    | { kind: "url"; url: string };
}

Stop reasons

A normalized enum of why a turn ended. Providers map their own values into this.

export type StopReason =
  | "end_turn"          // model finished naturally
  | "tool_use"          // model wants to call one or more tools
  | "max_tokens"        // hit the response cap
  | "stop_sequence"     // hit a configured stop sequence
  | "refusal"           // model declined
  | "error";            // provider-side error mapped to stop

Usage

export interface Usage {
  input_tokens: number;
  output_tokens: number;
  /** Tokens served from the provider's prompt cache. */
  cache_read_tokens?: number;
  /** Tokens written into the provider's prompt cache. */
  cache_creation_tokens?: number;
}

Model identification

skawld has no default model. A user must always provide one. The shape is a plain string (the provider knows how to validate it):

export type ModelId = string; // e.g. "claude-sonnet-4-6", "gpt-5", "deepseek-chat"

Events

A Run emits a stream of typed events. The event union is closed: no provider, tool, or extension may invent new event kinds. If you need a new event, update this doc first.

// src/core/events.ts

export type Event =
  | SystemEvent
  | AssistantEvent
  | UserEvent
  | PartialAssistantEvent
  | ToolCallStartEvent
  | ToolCallEndEvent
  | PermissionRequestEvent
  | UsageEvent
  | CompactionEvent
  | ResultEvent
  | ErrorEvent;

Event types in detail

/** Fired once at the start of a run with run-scoped metadata. */
export interface SystemEvent {
  type: "system";
  subtype: "init";
  session_id: string;
  run_id: string;
  model: ModelId;
  tools: string[];               // names of enabled tools
  permission_mode: PermissionMode;
  cwd: string;
}

/** A complete assistant message (text + thinking + tool_use blocks). */
export interface AssistantEvent {
  type: "assistant";
  message: Message;              // role = "assistant"
  stop_reason: StopReason;
}

/** A user-role message, including tool_result blocks the engine produced. */
export interface UserEvent {
  type: "user";
  message: Message;              // role = "user"
}

/** Streaming token deltas. Off by default; opt in with includePartialMessages. */
export interface PartialAssistantEvent {
  type: "partial_assistant";
  delta:
    | { kind: "text"; text: string }
    | { kind: "thinking"; text: string }
    | { kind: "tool_use_input"; tool_use_id: string; json_delta: string };
}

/** A tool call has been authorized and execution is starting. */
export interface ToolCallStartEvent {
  type: "tool_call_start";
  tool_use_id: string;
  tool_name: string;
  input: Record<string, unknown>;
}

/** A tool call has finished (success or error). */
export interface ToolCallEndEvent {
  type: "tool_call_end";
  tool_use_id: string;
  tool_name: string;
  is_error: boolean;
  duration_ms: number;
}

/** Permission decision required for one or more pending tool calls. */
export interface PermissionRequestEvent {
  type: "permission_request";
  requests: Array<{
    tool_use_id: string;
    tool_name: string;
    input: Record<string, unknown>;
    /** A short human-readable description from the tool (e.g. "Run: rm -rf node_modules"). */
    summary: string;
  }>;
}

/** Token usage for the turn that just completed. */
export interface UsageEvent {
  type: "usage";
  usage: Usage;
  /** Cumulative usage for the run so far. */
  cumulative: Usage;
}

/** Auto-compaction has summarized history. */
export interface CompactionEvent {
  type: "compaction";
  messages_before: number;
  messages_after: number;
  tokens_before: number;
  tokens_after: number;
  strategy: string;              // e.g. "default" or a custom strategy id
}

/** Terminal event for a run. */
export interface ResultEvent {
  type: "result";
  subtype: "success" | "aborted" | "error";
  stop_reason: StopReason;
  total_usage: Usage;
  duration_ms: number;
  /** The final assistant text, if any. */
  final_text?: string;
}

/** A typed error fired before a run terminates. May be followed by a result event. */
export interface ErrorEvent {
  type: "error";
  error: {
    name: string;                // matches SkawldError subclass name
    message: string;
    retryable: boolean;
    cause?: unknown;
  };
}

Event helpers

export function isAssistantEvent(e: Event): e is AssistantEvent {
  return e.type === "assistant";
}
// Provide one isXxxEvent helper per event kind. Useful in consumers.
Why a closed event union

An open event space means consumers have to write defensive code or risk silently dropping events. A closed union with a discriminator (type) gives TypeScript exhaustiveness checks and forces breaking changes to surface in compilation. Provider-specific extras should fit into existing events (e.g. a thinking block's signature), not new event kinds.

Error hierarchy

Always throw a subclass of SkawldError. Never throw a bare Error from anywhere in src/. Consumers catch on the class so they can branch on error type without string-matching.

// src/core/errors.ts

export class SkawldError extends Error {
  /** Stable, stringly-typed kind for log/UI surfaces. */
  abstract readonly kind: string;
  /** True if a retry is plausibly worth attempting. */
  readonly retryable: boolean = false;
  constructor(message: string, options?: { cause?: unknown }) {
    super(message, options);
    this.name = this.constructor.name;
  }
}

export class AuthError extends SkawldError {
  readonly kind = "auth";
  readonly retryable = false;
}

export class RateLimitError extends SkawldError {
  readonly kind = "rate_limit";
  readonly retryable = true;
  /** Seconds to wait before retrying, if the provider supplied one. */
  readonly retry_after_seconds?: number;
}

export class ContextLengthError extends SkawldError {
  readonly kind = "context_length";
  readonly retryable = false;   // triggers a compaction attempt, not a transport retry
}

export class PermissionDeniedError extends SkawldError {
  readonly kind = "permission_denied";
  readonly retryable = false;
  readonly tool_name: string;
  readonly reason: string;
}

export class ToolExecutionError extends SkawldError {
  readonly kind = "tool_execution";
  readonly retryable = false;
  readonly tool_name: string;
}

export class AbortError extends SkawldError {
  readonly kind = "abort";
  readonly retryable = false;
}

export class ProviderError extends SkawldError {
  readonly kind = "provider";
  /** HTTP status if the underlying transport returned one. */
  readonly status?: number;
  readonly retryable: boolean;
  constructor(message: string, opts: { status?: number; retryable: boolean; cause?: unknown }) {
    super(message, { cause: opts.cause });
    this.status = opts.status;
    this.retryable = opts.retryable;
  }
}

export class ConfigError extends SkawldError {
  readonly kind = "config";
  readonly retryable = false;
}
Tool execution errors are not thrown to consumers

When a tool throws or returns an error, the engine catches it, packages it as a tool_result with is_error: true, and feeds it back to the model so the model can recover. The ToolExecutionError class exists for the SDK's own consumption (e.g. for the optional onToolError hook later) and for the rare case where a tool throws something so catastrophic the engine cannot continue.

Retry policy (where errors come from)

The provider layer is responsible for retries at the transport level. Defaults:

  • Retry on RateLimitError (honor retry_after_seconds) and on ProviderError with retryable: true (typically HTTP 500/502/503/504 and network errors).
  • Exponential backoff: base 1s, max 30s, jitter ±20%.
  • Max attempts: 5 (configurable per Agent via maxRetries).
  • After the last attempt, the error propagates to the run as an ErrorEvent followed by a ResultEvent with subtype: "error".

AbortSignal

Every long-running operation accepts an AbortSignal. The Session owns one signal per active run; callers can supply their own which is chained.

// src/core/abort.ts

/**
 * Combine multiple AbortSignals into one. The result fires when any input fires.
 * Used to chain the consumer's signal with the internal run-scoped signal.
 */
export function anySignal(signals: (AbortSignal | undefined)[]): AbortSignal {
  const controller = new AbortController();
  for (const s of signals) {
    if (!s) continue;
    if (s.aborted) { controller.abort(s.reason); break; }
    s.addEventListener("abort", () => controller.abort(s.reason), { once: true });
  }
  return controller.signal;
}

/** Throw AbortError if the signal has fired. Useful at await boundaries. */
export function throwIfAborted(signal: AbortSignal): void {
  if (signal.aborted) {
    throw new AbortError("aborted", { cause: signal.reason });
  }
}

Cumulative usage helper

A tiny pure function used by the loop to accumulate per-turn usage into the run total. Keep it here so all modules use the same one.

export function addUsage(a: Usage, b: Usage): Usage {
  return {
    input_tokens: a.input_tokens + b.input_tokens,
    output_tokens: a.output_tokens + b.output_tokens,
    cache_read_tokens: (a.cache_read_tokens ?? 0) + (b.cache_read_tokens ?? 0),
    cache_creation_tokens: (a.cache_creation_tokens ?? 0) + (b.cache_creation_tokens ?? 0),
  };
}

ID generation

Use crypto.randomUUID() (built into Node 20+) for session ids, run ids, and any other internally generated ids. No external dependency needed.

Build checklist

  • Create src/core/types.ts with Message, ContentBlock (all five kinds), StopReason, Usage, ModelId, PermissionMode.
  • Create src/core/events.ts with all 11 event interfaces and the Event union. Add one isXxxEvent helper per kind.
  • Create src/core/errors.ts with SkawldError base and all 8 subclasses.
  • Create src/core/abort.ts with anySignal and throwIfAborted.
  • Add addUsage to src/core/types.ts (or a sibling usage.ts).
  • Write a vitest file per module that exercises the shape of each type and that every error subclass sets name and kind correctly.
  • Re-export everything from src/core/index.ts.
