Agent & loop

Where everything comes together: the Agent class, the Session class, the model→tool orchestration loop, parallel tool execution, streaming, abort handling, and 80%-threshold compaction. All other modules feed into this one.

The Agent class

// src/core/agent.ts
import type { BaseProvider } from "../providers/base";
import type { ToolRegistry } from "../tools/registry";
import type { SessionStore } from "../sessions/store";
import type { PermissionMode, PermissionRule, CanUseTool } from "../permissions/engine";
import type { CompactionStrategy } from "./compaction";
import type { ModelId } from "./types";

export interface AgentOptions {
  /** LLM provider. Required. */
  provider: BaseProvider;
  /** Model id string. Required (skawld has no default). */
  model: ModelId;
  /** Tool registry. If omitted, defaults to the built-in tools. */
  tools?: ToolRegistry;
  /** Permission configuration. */
  permissions?: {
    mode?: PermissionMode;           // default "default"
    rules?: PermissionRule[];        // default []
    canUseTool?: CanUseTool;
  };
  /** Where to persist sessions. Defaults to SqliteSessionStore at .skawld/sessions.db. */
  sessionStore?: SessionStore;
  /** Working directory for tools. Defaults to process.cwd(). */
  cwd?: string;
  /** Additional system-prompt text appended after the default. */
  systemPrompt?: string;
  /** Override compaction. Defaults to the built-in strategy at 80% of context window. */
  compaction?: CompactionStrategy;
  /** Max retries for retryable provider errors. Default 5. */
  maxRetries?: number;
  /** Max output tokens per turn. Default 8192 (or model max if smaller). */
  maxOutputTokens?: number;
  /** Emit partial_assistant events with token deltas. Default false. */
  includePartialMessages?: boolean;
  /** Hard cap on turns per run, to prevent runaway loops. Default 100. */
  maxTurns?: number;
}

export class Agent {
  constructor(public readonly opts: AgentOptions) { /* validates and stores */ }

  /** Create or resume a session. */
  async session(input?: { id?: string; meta?: Record<string, unknown> }): Promise<Session> { /* ... */ }

  /** Release resources (closes the session store). */
  async close(): Promise<void> { /* ... */ }
}

The Agent is immutable after construction. To change provider, model, or other configuration, build a new Agent. Sessions persist across Agents because they live in the SessionStore.

The Session class

// src/core/session.ts

export interface RunOptions {
  /** Caller-provided signal; chained with the session's internal one. */
  signal?: AbortSignal;
  /** Per-run override of max output tokens. */
  maxOutputTokens?: number;
  /** Per-run override of the temperature. */
  temperature?: number;
  /** Attach images to the user prompt. */
  images?: Array<{ data: string; mediaType: string } | { url: string }>;
}

export class Session {
  readonly id: string;
  readonly createdAt: Date;
  readonly meta: Record<string, unknown>;

  /** Total number of messages in the session, including those from previous runs. */
  get messageCount(): number;

  /** Run the agent with a new user prompt. Returns an async iterator of Event. */
  run(prompt: string, opts?: RunOptions): AsyncIterable<Event>;

  /**
   * Cancel the currently running iteration. The next event yielded will be a
   * ResultEvent with subtype "aborted". Idempotent.
   */
  abort(reason?: unknown): void;

  /** Update metadata in the store. */
  updateMeta(patch: Record<string, unknown>): Promise<void>;
}

A session can have only one active run at a time. Calling run() while another is in flight throws a ConfigError. The intended usage is "await the async iterator to completion (or call abort()) before starting another run".

The loop

The loop drives a single run() call. It is implemented as an async function* that yields Events and terminates when the model stops calling tools (or hits an error / abort / turn limit).

┌──────────────────────────────────────────────────────────────────────┐ │ runLoop(session, prompt) │ └──────────────────────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────────────────────┐ │ yield SystemEvent { init } │ │ append user message → store + in-memory │ │ yield UserEvent │ └──────────────────────────────────────────────────────────────────────┘ │ ▼ (start of turn) ┌──────────────────────────────────────────────────────────────────────┐ │ if shouldCompact() → run compaction strategy, append result │ │ build ProviderRequest (system, tools, messages) │ │ open provider.stream(req) with retry on transient errors │ │ │ │ for each ProviderStreamEvent: │ │ - text_delta / thinking_delta → buffer; emit partial_assistant │ │ - tool_use_start/_delta/_end → assemble tool_use blocks │ │ - message_end → finalize assistant Message and Usage │ │ │ │ yield AssistantEvent { message, stop_reason } │ │ append assistant message → store │ │ yield UsageEvent { usage, cumulative } │ └──────────────────────────────────────────────────────────────────────┘ │ ┌────────┴─────────┐ │ stop_reason? │ └────────┬─────────┘ │ "tool_use" │ "end_turn" / "max_tokens" / etc. ▼ ▼ ┌────────────────────────────────────────┐ ┌─────────────────────────┐ │ scheduler: │ │ yield ResultEvent │ │ reads[] in parallel │ │ stop & exit │ │ then writes/execs sequentially │ └─────────────────────────┘ │ permission engine resolves each call │ │ (PermissionRequestEvent if "ask") │ │ for each call: │ │ yield ToolCallStartEvent │ │ try: tool.execute(input, ctx) │ │ catch: synthesize is_error result │ │ yield ToolCallEndEvent │ │ │ │ assemble user-role Message containing │ │ all tool_result blocks (one per call) │ │ append → store │ │ yield UserEvent { the tool_result msg }│ └──────────────────┬─────────────────────┘ │ └──▶ (next turn)

Pseudocode

export async function* runLoop(
  session: SessionInternal,
  prompt: string,
  opts: RunOptions,
): AsyncGenerator<Event> {
  const runId = crypto.randomUUID();
  const signal = anySignal([session.internalSignal, opts.signal]);

  const startedAt = Date.now();
  let totalUsage: Usage = zeroUsage();

  yield { type: "system", subtype: "init", session_id: session.id, run_id: runId,
          model: agent.opts.model, tools: agent.tools.list().map(t => t.name),
          permission_mode: agent.permissions.mode, cwd: agent.opts.cwd };

  // Append the user prompt (possibly with images) and emit UserEvent.
  const userMsg = buildUserMessage(prompt, opts.images);
  await session.append([userMsg]);
  yield { type: "user", message: userMsg };

  for (let turn = 0; turn < agent.opts.maxTurns; turn++) {
    throwIfAborted(signal);

    // Compaction check (see "Compaction" section).
    if (await maybeCompact(session, agent, signal)) {
      const c = session.lastCompactionInfo!;
      yield { type: "compaction", ...c };
    }

    // Build request.
    const req: ProviderRequest = buildRequest(session, agent, opts, signal);

    // Stream the turn.
    const { assistantMessage, stopReason, usage } =
      await streamTurn(agent.provider, req, /* yield */ (e) => { /* PartialAssistantEvent */ });

    // Persist + emit.
    await session.append([assistantMessage]);
    yield { type: "assistant", message: assistantMessage, stop_reason: stopReason };
    totalUsage = addUsage(totalUsage, usage);
    yield { type: "usage", usage, cumulative: totalUsage };

    if (stopReason !== "tool_use") {
      yield { type: "result", subtype: "success", stop_reason: stopReason,
              total_usage: totalUsage, duration_ms: Date.now() - startedAt,
              final_text: extractFinalText(assistantMessage) };
      return;
    }

    // Execute tool calls (see "Scheduler").
    const toolUseBlocks = assistantMessage.content.filter(isToolUseBlock);
    const resultBlocks = yield* executeToolCalls(toolUseBlocks, agent, session, signal);

    // Append all results as a single user message.
    const userResultMsg: Message = { role: "user", content: resultBlocks };
    await session.append([userResultMsg]);
    yield { type: "user", message: userResultMsg };
  }

  // Hit the turn cap.
  yield { type: "error", error: { name: "TurnLimitError", message: "max turns exceeded", retryable: false } };
  yield { type: "result", subtype: "error", stop_reason: "error",
          total_usage: totalUsage, duration_ms: Date.now() - startedAt };
}

Tool scheduler

Per the rules in 02 — Tools: read-scope tools run in parallel; write-scope and exec-scope tools run sequentially in emission order.

// src/core/scheduler.ts

export async function* executeToolCalls(
  blocks: ToolUseBlock[],
  agent: AgentInternal,
  session: SessionInternal,
  signal: AbortSignal,
): AsyncGenerator<Event, ToolResultBlock[]> {
  // 1. Resolve every block to a (Tool, validatedInput, summary).
  //    If a tool is unknown, synthesize an is_error result immediately (do not throw).
  const resolved = blocks.map((b) => resolveCall(b, agent));

  // 2. Bucket by scope.
  const reads = resolved.filter((r) => r.tool.scope === "read");
  const writes = resolved.filter((r) => r.tool.scope !== "read");

  // 3. Permission pass: evaluate rules + mode for every call (sync, fast).
  //    Collect calls that resolve to "ask"; emit a single PermissionRequestEvent
  //    for the whole turn.
  const decisions = await resolveAllPermissions(resolved, agent, signal, /* yield */ (e) => { /* ... */ });

  // 4. Execute reads concurrently.
  const readResults = await Promise.all(reads.map(async (r) => {
    yield { type: "tool_call_start", tool_use_id: r.id, tool_name: r.tool.name, input: r.input };
    const result = await safeExecute(r, agent, session, signal, decisions);
    yield { type: "tool_call_end", tool_use_id: r.id, tool_name: r.tool.name, is_error: !!result.is_error, duration_ms: result.duration_ms };
    return toToolResultBlock(r, result);
  }));

  // 5. Execute writes/execs sequentially.
  const writeResults: ToolResultBlock[] = [];
  for (const r of writes) {
    throwIfAborted(signal);
    yield { type: "tool_call_start", tool_use_id: r.id, tool_name: r.tool.name, input: r.input };
    const result = await safeExecute(r, agent, session, signal, decisions);
    yield { type: "tool_call_end", tool_use_id: r.id, tool_name: r.tool.name, is_error: !!result.is_error, duration_ms: result.duration_ms };
    writeResults.push(toToolResultBlock(r, result));
  }

  // 6. Return results in the SAME ORDER as the original tool_use blocks.
  return reorderToMatch(blocks, [...readResults, ...writeResults]);
}
async generators yielding events from within Promise.all

The pseudocode above is sketchy when it shows yield inside a Promise.all. In real TypeScript, that does not work directly — generators yield serially. The implementation should buffer events from concurrent reads in a small queue and yield them in the order they happen; the caller cannot reliably distinguish strict per-call ordering for reads, only that reads happen before writes. The contract is: tool_call_start always precedes the corresponding tool_call_end for the same tool_use_id, and all read events precede all write events.

safeExecute

async function safeExecute(call, agent, session, signal, decisions) {
  const decision = decisions[call.id];
  const t0 = Date.now();
  if (decision.decision === "deny") {
    return { content: `Tool call denied: ${decision.reason}`, is_error: true,
             duration_ms: Date.now() - t0, summary: call.summary };
  }
  try {
    const ctx: ToolContext = {
      cwd: agent.opts.cwd, signal, fileReadTracker: session.fileReadTracker,
      sessionId: session.id, runId: session.activeRunId!,
      sessionStore: agent.sessionStore,
    };
    // Apply input rewriting if the canUseTool callback provided updatedInput.
    const input = decision.updatedInput ?? call.input;
    const result = await call.tool.execute(input, ctx);
    return { ...result, duration_ms: Date.now() - t0 };
  } catch (err) {
    if (err instanceof AbortError) throw err;            // propagate
    // Any other error becomes an is_error tool_result. The model can recover.
    return { content: `Tool failed: ${err.message}`, is_error: true,
             duration_ms: Date.now() - t0, summary: call.summary };
  }
}

Streaming and partial events

  • The provider always streams. Inside the loop, deltas are accumulated into the assistant Message while also being yielded as PartialAssistantEvent when includePartialMessages is enabled.
  • Tool inputs are streamed as JSON deltas via tool_use_input_delta. The loop accumulates the full JSON, parses it on tool_use_end, and stores it as a ToolUseBlock.input. Partial events for tool input are forwarded when partial mode is on.
  • If JSON parsing of a finalized tool input fails, the call is converted into an immediate is_error tool_result — never thrown.

Abort handling

  • session.abort(reason) aborts the internal controller. The next await boundary in the loop checks the signal and throws AbortError.
  • AbortError is caught at the top of the loop. The loop yields a final ResultEvent with subtype: "aborted" and exits.
  • The currently running provider stream is canceled by the chained signal. Tools receive the same signal and must stop ASAP.
  • If abort fires after a tool call started but before it returned, that tool's tool_call_end is emitted with is_error: true and a message indicating abort. The assistant message is still emitted; the loop terminates before the next turn.

Compaction

Compaction prevents context overflow by replacing the older half of the message history with a model-generated summary when the conversation grows to 80% of the model's context window.

Trigger

// Computed before each turn.
const limit = agent.provider.contextWindow(agent.opts.model);
const projected = lastUsage.input_tokens + agent.opts.maxOutputTokens;
if (projected >= 0.8 * limit) {
  await runCompaction(...);
}

lastUsage.input_tokens comes from the most recent UsageEvent. If no run has happened yet, the trigger does not fire. Token estimates are API-reported; skawld does not run a local tokenizer.

Default strategy

// src/core/compaction.ts

export interface CompactionContext {
  messages: Message[];        // full session message history
  provider: BaseProvider;
  model: ModelId;
  signal: AbortSignal;
}

export interface CompactionStrategy {
  /** Stable identifier surfaced in CompactionEvent.strategy. */
  readonly id: string;
  /**
   * Produce a new message array that replaces the input.
   * Should preserve the most recent N turns intact and condense the rest.
   */
  compact(ctx: CompactionContext): Promise<Message[]>;
}

export const defaultCompaction: CompactionStrategy = {
  id: "default-keep-recent-10",
  async compact({ messages, provider, model, signal }) {
    // 1. Always keep the last 10 *turn boundaries* (assistant + user pair counts as one boundary).
    // 2. Send the OLDER prefix to the model with a system prompt asking for a
    //    structured summary: decisions made, files touched, open questions, and current task state.
    // 3. Replace the older prefix with a single synthetic user message:
    //    {
    //      role: "user",
    //      content: [{ type: "text", text: "<summary of earlier conversation>\n\n${summary}" }]
    //    }
    // 4. Return [summaryMessage, ...recentMessages].
    // 5. Use a non-streaming variant of provider.stream (just consume the stream and assemble).
  },
};

Overriding compaction

const agent = new Agent({
  // ...
  compaction: {
    id: "my-strategy",
    async compact(ctx) {
      // Custom logic: maybe summarize per-file, maybe drop based on relevance, etc.
      return shortenedMessages;
    },
  },
});

Event

After compaction runs, a CompactionEvent is yielded with before/after counts. The replaced messages are not deleted from the session store — the store retains them so a future "show full history" view can reconstruct them — but the in-memory state used by subsequent turns reflects the compacted form.

Implementation note: store divergence

To make compaction safe to rerun, the in-memory view of "messages we send to the provider" must diverge from "messages stored in the SessionStore". Track them separately on the Session: session.providerView (potentially compacted) vs. session.fullHistory (always all messages). The compaction event records before/after counts of providerView.

Error handling in the loop

  • Retryable provider errors (RateLimitError, ProviderError with retryable=true): handled inside provider.stream by the withRetry helper. If retries are exhausted, the error escapes stream().
  • Escaped provider errors: emit an ErrorEvent, then a ResultEvent with subtype: "error", then return.
  • ContextLengthError: trigger a one-shot compaction (independent of the 80% threshold) and retry the turn once. If it happens again, surface as an error.
  • Tool errors: never escape. Synthesized as tool_result is_error; loop continues.
  • AbortError: caught at the top; yield aborted result and return.
  • Anything else (programmer error, bug): emit ErrorEvent with the original message wrapped; emit ResultEvent error; return. Do not swallow.

SDK consumption shape

const agent = new Agent({ /* ... */ });
const sess = await agent.session();

for await (const event of sess.run("fix the failing test")) {
  switch (event.type) {
    case "assistant":          /* render text/thinking/tool_use */ break;
    case "tool_call_start":    /* show "Running Bash: npm test" */ break;
    case "tool_call_end":      /* show duration / error */ break;
    case "permission_request": /* prompt UI */ break;
    case "usage":              /* update cost meter */ break;
    case "compaction":         /* notify user */ break;
    case "result":             /* done */ return;
    case "error":              /* surface error UI */ break;
  }
}

Callers may abandon the iterator without finishing; in that case the Session keeps internal state but the run is effectively suspended. To cancel cleanly, call sess.abort() first.

Build checklist

  • src/core/agent.ts with Agent class and AgentOptions.
  • src/core/session.ts with Session class, run() async generator, abort(), single-active-run invariant.
  • src/core/loop.ts with runLoop implementing the full event flow: system → user → (per turn: maybe compact → assistant → usage → tool calls → user) → result.
  • src/core/scheduler.ts with parallel-reads-then-sequential-writes execution; permission integration; aggregated PermissionRequestEvent; per-call canUseTool.
  • src/core/compaction.ts with CompactionStrategy interface and the default 10-recent-turns strategy.
  • 80% threshold compaction trigger using API-reported usage.
  • One-shot retry on ContextLengthError.
  • Abort propagation through provider, tools, and any in-flight permission callback.
  • Max-turn cap (default 100) yielding an error result.
  • Single active run per Session enforcement.
  • Divergent in-memory providerView vs fullHistory; SessionStore always receives the full history.
  • Tests with a mock provider and mock tools that exercise: simple text turn, single tool call, multiple parallel reads, mixed read+write, permission ask/allow/deny, abort mid-stream, compaction trigger, context-length recovery, error escape.
